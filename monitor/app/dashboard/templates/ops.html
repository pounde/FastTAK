Operations

Certificates

Create Client Certificate ?Generate a client certificate (.pem + .p12) for a TAK device or user. The .p12 is re-exported with modern AES-256-CBC ciphers. Download from the host with ./certs.sh download <name>.p12. 

        


        
        

            
Create Server Certificate
                ?Generate a server certificate for a TAK Server instance. Typically only needed when adding a new server or after CA rotation.
            

            

                
                
            

            

        


        
        

            
Revoke Certificate
                ?Permanently revoke a client or server certificate. Revoked certs are added to the CRL and rejected by TAK Server. This cannot be undone.
            

            

                
                
            

            

        


        
        

            
Certificate List
                ?List all certificate files (.pem, .p12, .jks, .csr) in the TAK Server cert directory.

VACUUM FULL
            ?Locks the database for the duration of the operation. Use only to reclaim disk space after large deletions (e.g., after enabling retention for the first time). All TAK clients will be unable to send or receive data while this runs.
        

        

            
        

        

    











    
Alert Testing


    


        
        

            
Test Email Alert
                ?Send a test email to the configured ALERT_EMAIL address via SMTP. Verifies alerting works end-to-end. Configure SMTP_HOST, SMTP_USER, SMTP_PASSWORD, and ALERT_EMAIL in .env.
            

            

                
            

            

        


        
        

            
Test SMS Alert
                ?Send a test SMS to the configured SMS_TO numbers via Twilio or Brevo. Configure SMS_PROVIDER, SMS_API_KEY, SMS_FROM, and SMS_TO in .env.
